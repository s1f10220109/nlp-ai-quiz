🎨 1. UI/UXの基礎概念・認知心理学（第1・2回）

ユーザーが製品・サービスを利用する過程（利用前・中・後）のすべての体験。
答え UX (User Experience)
ユーザーとシステムの接点（画面デザイン、操作部など）。
答え UI (User Interface)
ISO 9241-210で定義された、ユーザーのニーズや能力に合わせて設計するプロセス。
答え 人間中心設計 (HCD)
事物が持つ「どのような行為が可能か」という性質（例：椅子は座れる）。
答え アフォーダンス
ユーザーに適切な行動を伝えるための、知覚可能な手がかり（マーク、音など）。
答え シグニファイア
操作部と対象の動きや配置の対応関係（例：右のスイッチで右のライトがつく）。
答え 対応づけ (Mapping)
行為の結果をユーザーに伝えること（例：ボタンを押すと音が鳴る）。
答え フィードバック
システムがどう動くかについて、ユーザーの頭の中に形成されるイメージや理解。
答え メンタルモデル
システムの仕組みそのものの構造や論理。
答え 概念モデル (Design model)
意図（ゴール）は正しいが、実行段階でやり損なうエラー（押し間違いなど）。
答え スリップ
そもそも意図（ゴール）や計画自体が間違っているエラー（思い込みなど）。
答え ミステーク
人間の短期記憶の容量限界とされる数（7±2）。
答え マジカルナンバー7
情報を意味のあるまとまりにして記憶しやすくする手法。
答え チャンク化
「ユーザーを騙す」目的で設計された悪意あるUIデザイン。
答え ダークパターン
ニールセンのユーザビリティ10原則の一つ。システムの状態を常に知らせるべきという原則。
答え システム状態の視認性
過去の経験や慣習に基づいて、学習しなくても使えること。
答え 学習のしやすさ (Learnability)
ユーザーがエラーをした際に、容易に回復できること。
答え エラー回復性

🔍 2. デザインプロセス・調査手法（第2回）

HCDプロセスの4要素：①利用状況の把握、②(A)、③解決策の作成、④評価。(A)は？
答え ユーザーの要求事項の明示
調査対象のコミュニティに入り込み、行動を詳細に観察する定性調査手法。
答え エスノグラフィー
平均的なユーザーではなく、極端な行動をとるユーザー（ヒントを得やすい）。
答え エクストリームユーザー
架空の典型的ユーザー像。名前、年齢、性格などを詳細に設定したもの。
答え ペルソナ
ユーザーの行動、思考、感情を時系列に沿って可視化したマップ。
答え カスタマージャーニーマップ
ユーザーの行動から「心の声」を推測し、背後にある「価値」を導き出す手法。
答え KA法
ブレインストーミングの4原則：①質より量、②粗い考え歓迎、③結合・改善、あと一つは？
答え 批判厳禁
アイデア発想のためのチェックリスト（転用、結合、変更、拡大、縮小など）。
答え オズボーンのチェックリスト
完成品ではなく、試作段階の模型を使って検証すること。
答え プロトタイピング
紙に画面を描いて検証する、低コストなプロトタイプ手法。
答え ペーパープロトタイピング
ユーザーに価値を提供できる「必要最小限の機能」を持った製品・サービス。
答え MVP (Minimum Viable Product)
2つの異なるデザイン（AとB）を比較し、どちらが優れているかを定量的に評価するテスト。
答え A/Bテスト
被験者にタスクを実行してもらい、行動や発話を観察する評価手法。
答え ユーザビリティテスト
被験者が頭に浮かんだことを声に出しながら操作する手法。
答え 発話思考法 (Think Aloud)
専門家が経験則（ヒューリスティックス）に基づいてUIを評価する手法。
答え ヒューリスティック評価

🖥️ 3. インタフェースの歴史（第3回）

1945年にヴァネヴァー・ブッシュが構想した、情報検索システムの概念。
答え Memex (メメックス)
1963年にサザランドが開発した、対話的図形処理システムの元祖。ライトペンを使用。
答え Sketchpad
1968年にエンゲルバートがデモを行ったシステム。マウス、マルチウィンドウ等を発表。
答え NLS (oN-Line System)
エンゲルバートが発明した、X-Y座標を入力するポインティングデバイス。
答え マウス
1972年にアラン・ケイが提唱した、子供がプログラミングできる理想のPC構想。
答え Dynabook
アラン・ケイらがゼロックスPARCで開発した、GUIを備えた最初期のワークステーション。
答え Alto (アルト)
コンピュータの操作対象を、現実世界の物（机、ゴミ箱、フォルダ）になぞらえる考え方。
答え メタファ (Desktop Metaphor)
画面上の対象をマウス等で直接的につかんだり操作したりする感覚のUI概念。
答え 直接操作 (Direct Manipulation)
画面上の見た目と印刷結果が一致する性質のこと（What You See Is What You Get）。
答え WYSIWYG
ウィンドウ、アイコン、メニュー、ポインティングデバイスを用いたUIの総称。
答え WIMP / GUI

⌨️ 4. 入力技術・日本語入力（第4・5回）

現在最も普及している英字キーボード配列。タイプライターのジャム防止に由来。
答え QWERTY配列
入力効率を最適化するために考案されたが、普及しなかったキーボード配列。
答え Dvorak配列
複数のキーを同時に押すことで文字を入力する方式（速記などで利用）。
答え コード入力 (Chord)
1978年に東芝が初めて実用化した、日本語入力に不可欠な技術。
答え かな漢字変換
ガラケーで使われた、同じキーを何度も押して文字を切り替える入力方式。
答え トグル入力 (マルチタップ)
スマートフォンで主流の、キーをタッチして上下左右にスライドさせる入力方式。
答え フリック入力
キーボードのキー配置において、物理的な突起などでホームポジションを示す工夫。
答え タクタイル (触覚的な目印)

🌐 5. Web技術の基礎（第6回）

Webページを記述するためのマークアップ言語。ページの「構造」を担う。
答え HTML
Webページの「見た目・レイアウト・装飾」を指定する言語。
答え CSS
Webページ上で動的な処理を行うためのプログラミング言語。
答え JavaScript
インターネット上でWebページを転送するための通信プロトコル。
答え HTTP / HTTPS
Web上のリソース（ページや画像）の場所を示すアドレス。
答え URL
インターネット上の住所にあたるIPアドレスを、人間が覚えやすい名前に変えたもの。
答え ドメイン名
ドメイン名とIPアドレスを変換するシステム。
答え DNS
Webページが見つからない場合にサーバーが返すHTTPステータスコード。
答え 404 (Not Found)
リクエストが成功したことを示すHTTPステータスコード。
答え 200 (OK)
サーバー内部でエラーが発生したことを示すHTTPステータスコード。
答え 500 (Internal Server Error)
コンテンツを高速に配信するために、世界中に分散配置されたサーバー網。
答え CDN
画面サイズ（PC、スマホ）に応じてCSSを切り替え、レイアウトを調整する手法。
答え レスポンシブWebデザイン
レスポンシブデザインを実現するためにCSSで使われる機能（画面幅で条件分岐）。
答え メディアクエリ
障害の有無や年齢にかかわらず、誰もがWebを利用できるようにすること。
答え Webアクセシビリティ
画像が表示できない環境のために設定すべきHTML属性（代替テキスト）。
答え alt属性

💻 6. プログラミング・非同期通信（第7・8回）

HTMLの要素をプログラムから操作するための仕組み（木構造モデル）。
答え DOM (Document Object Model)
ユーザーの操作（クリックなど）をきっかけに処理を実行するプログラミング方式。
答え イベント駆動 (Event Driven)
ボタンがクリックされた時に実行される関数のこと。
答え イベントリスナー / ハンドラ
JavaScriptで「再代入可能な」変数を宣言するキーワード（ES6以降）。
答え let
JavaScriptで「再代入不可能な」変数を宣言するキーワード（ES6以降）。
答え const
関数を短く記述するための記法（ES6以降）。 (x) => x * 2
答え アロー関数
ページ全体を再読み込みせず、サーバーと非同期通信して一部だけ更新する技術。
答え Ajax
Ajaxなどでデータ交換によく使われる、軽量なテキストデータ形式。
答え JSON
キーと値のペアで記述されるJSONの、キーを囲む記号は？
答え ダブルクォーテーション ( " )
JavaScriptで非同期通信を行うための現代的なAPI（XMLHttpRequestの後継）。
答え Fetch API
非同期処理の完了または失敗を表すJavaScriptのオブジェクト。
答え Promise
1つのWebページ内でアプリのような挙動を実現する構成（遷移時にリロードしない）。
答え SPA (Single Page Application)
複数のWebサービスやAPIを組み合わせて、新しいサービスを作ること。
答え マッシュアップ
Pythonの標準的なGUIツールキット。
答え Tkinter
GUIプログラムで、イベントの発生を待ち続ける無限ループ処理。
答え メインループ (mainloop)

📊 7. 情報可視化・アニメーション（第9・10回）

抽象的なデータを視覚的に表現し、人間の認知を支援すること。
答え 情報可視化
情報をわかりやすく図解した、視覚的魅力重視の表現（可視化との対比）。
答え インフォグラフィクス
ナポレオンのロシア遠征の損失を視覚化した有名な図の作者。
答え Minard (ミナール)
シュナイダーマンの可視化のマントラ。「まずは(A)、次にズーム＆フィルタ、詳細はオンデマンド」。
答え 全体 (Overview first)
人間が図形をグループとして知覚する法則（近接、類同、閉合など）。
答え ゲシュタルト法則
不要な装飾やインクの無駄遣いを指す、エドワード・タフテの用語。
答え チャートジャンク
階層構造（木構造）を、入れ子の長方形の面積で表現する可視化手法。
答え Treemap
単語の出現頻度などを文字の大きさで表現する可視化手法。
答え タグクラウド (Word Cloud)
詳細を見せつつ、全体の文脈も同時に表示する手法（魚眼レンズなど）。
答え Focus + Context
画像を「点の集合」で表現する形式。拡大すると画質が荒くなる。
答え ビットマップ (ラスター)
画像を「数式や座標」で表現する形式。拡大しても劣化しない。
答え ベクター
アニメーションの動き出しや停止時に速度を変化させ、自然な動きにする技法。
答え イージング (Slow In, Slow Out)
1秒間に表示されるフレーム数。アニメーションの滑らかさの指標。
答え fps (frames per second)
ボタンのON/OFFなど、微細だが操作感を向上させる小さなアニメーションや反応。
答え マイクロインタラクション

🔎 8. 情報検索・レコメンデーション（第11回）

検索結果の中に含まれる「正解（適合文書）」の割合（精度）。
答え 適合率 (Precision)
世の中にある全ての正解のうち、検索でヒットした割合（網羅率）。
答え 再現率 (Recall)
適合率と再現率の調和平均をとった指標。
答え F値
文書内の単語の重要度を「頻度」と「希少性（逆文書頻度）」で評価する手法。
答え tf-idf法
Web上のリンク構造（被リンク数）に基づいてページの重要度を測るGoogleの初期アルゴリズム。
答え PageRank
Web上の文書を自動で巡回して収集するプログラム。
答え クローラ (ロボット)
「この商品に似た商品はこれ」と、アイテムの特徴に基づいて推薦する方式。
答え コンテンツベースフィルタリング
「この商品を買った人はこれも買っています」と、他者の行動履歴を利用する推薦方式。
答え 協調フィルタリング
自分に都合の良い情報ばかりが表示され、視野が狭くなる現象。
答え フィルターバブル
偶然に素敵なものや有益な情報を発見する現象・能力。
答え セレンディピティ

🥽 9. VR/AR・CSCW・IoT（第12〜14回）

コンピュータで作られた仮想空間に没入する技術。
答え VR (Virtual Reality)
現実空間にデジタル情報を重ね合わせて表示する技術。
答え AR (Augmented Reality)
現実世界と仮想世界を融合させる技術全般（VR/ARを含む概念）。
答え MR (Mixed Reality) / xR
VR体験などで得られる「実在感」や「没入感」のこと。
答え イマージョン / プレゼンス
視覚と平衡感覚のズレなどが原因で起こる、乗り物酔いに似た症状。
答え VR酔い (Cyber sickness)
レーザー光を使って対象までの距離や形状を測定するセンサー技術。
答え LiDAR
振動や力によって、皮膚感覚（触覚）をフィードバックする技術。
答え ハプティクス
インターネット上の仮想空間において、自分の分身となるキャラクター。
答え アバター
コンピュータを用いた協調作業を研究する分野。
答え CSCW
複数人が協調作業を行うためのソフトウェア（Slack, Teamsなど）。
答え グループウェア
時間も場所も異なる場合のコミュニケーション（メール、Wikiなど）。
答え 非同期・分散
ビデオ会議のように、同じ時間を共有するコミュニケーション。
答え 同期型
遠隔地にいながら、あたかもその場にいるかのような感覚を与える技術。
答え テレプレゼンス
組織の生産性向上に重要とされる、休憩時の雑談などのコミュニケーション。
答え インフォーマルコミュニケーション
あらゆる「モノ」がインターネットにつながる仕組み。
答え IoT (Internet of Things)
コンピュータが環境に溶け込み、いつでもどこでも利用できる概念（M.ワイザー提唱）。
答え ユビキタスコンピューティング
ユーザーの注意を過度に引かず、生活に溶け込む情報の提示のあり方。
答え カーム・テクノロジー
デジタル情報を、実体のある物（物理的アイコンなど）を使って操作するUI。
答え タンジブルUI
センサーやモーターなどの電子部品とソフトウェアを組み合わせた創作活動。
答え フィジカルコンピューティング
クラウドではなく、端末（デバイス）の近くでデータ処理を行う方式。
答え エッジコンピューティング
ロボット工学三原則を提唱したSF作家。
答え アイザック・アシモフ
テクノロジーを使って人間の身体能力や知覚能力を強化・拡張すること。
答え 人間拡張 (Human Augmentation)

⚙️ 10. コード読解・応用問題対策

setInterval(func, 1000) の動作は？
答え 1秒（1000ms）ごとに func を繰り返し実行する。
setTimeout(func, 1000) の動作は？
答え 1秒後に1回だけ func を実行する。
localStorage と sessionStorage の違いは？
答え localは永続保存、sessionはブラウザ(タブ)を閉じると消える。
document.getElementById("hoge") の役割は？
答え IDが "hoge" のHTML要素を取得する。
document.querySelector(".box") の役割は？
答え クラス名 "box" を持つ最初の要素を取得する。
サーバー側での処理（PageRank計算など）に適している言語の例は？
答え Python
10進数の「10」は、16進数では何？
答え A
10進数の「15」は、16進数では何？
答え F
色の三属性：色相、明度、あと一つは？
答え 彩度
デジタルツインとは？
答え 現実世界の環境をデジタル空間に双子のように再現し、シミュレーションする技術。
協調フィルタリングで、AさんとBさんの好みが「似ている」と判断する計算原理は？
答え 購入履歴などをベクトルと見なし、類似度（相関係数やコサイン類似度）を計算する。
